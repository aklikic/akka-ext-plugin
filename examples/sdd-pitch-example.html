Specification-Driven Development

Go from a one-line idea to a production-ready Akka service in five structured steps. Each step produces a versioned artifact that feeds the next.

Step 1

/specify — capture the what

Describe your feature in plain language. The AI generates a structured specification with user stories, acceptance scenarios, requirements, and success criteria.

Produces:  spec.md + spec-diagrams.md
Input — your one-liner
"Build a loan underwriting service with two-stage AI pipeline
(Compliance Check + Credit Check), queryable status, and a simple UI."
Output — spec.md (excerpt)
### User Story 2 — Automated Underwriting Pipeline (P1)

Once submitted, the system runs a two-stage AI-powered pipeline.
First, a Compliance Check screens against a blocked list.
If passed, a Credit Check evaluates the financial profile.

Acceptance Scenarios:

1. Given a non-blocked applicant with score 750, DTI 25%, LTI 3x
   When the pipeline runs
   Then compliance passes, credit = LOW risk → APPROVED

2. Given an applicant on the blocked list
   When the pipeline runs
   Then compliance fails, no credit check → DECLINED

3. Given a non-blocked applicant with score 650, DTI 38%, LTI 5x
   When the pipeline runs
   Then credit = MEDIUM risk → REFERRED (awaits review)

Step 2

/clarify — fill the gaps

The AI reads the spec, identifies up to 5 underspecified areas, asks targeted questions, and encodes answers back into the spec.

Produces:  updated spec.md (gaps filled)
Example — clarification Q&A
Q1 — How should credit bureau data be sourced?
Answer: Use stub services returning deterministic data by applicant name. The credit agent fetches score and debt via @FunctionTool, not from the form.
Q2 — What happens when a guardrail blocks an agent interaction?
Answer: The compliance step catches the guardrail exception directly and declines with a clear explanation — no retries wasted.
Q3 — Should MEDIUM-risk applications auto-expire?
Answer: No. The workflow stays in REFERRED status indefinitely until a reviewer acts. No automatic timeout initially.
Step 3

/plan — design the how

Translates the spec into Akka component architecture: SDK primitives, data model, interactions, project structure, and key design decisions. Verified against the Akka constitution.

Produces:  plan.md + plan-diagrams.md + data-model.md + research.md + contracts/
Output — plan.md (excerpt)
## Component Design

### UnderwritingWorkflow (Workflow)
- Extends Workflow<LoanApplication>
- Holds all application state — no separate entity needed
- Steps: complianceStep → creditStep → decisionStep
- Settings: defaultStepTimeout(120s), maxRetries(2)

### ComplianceCheckAgent (Agent)
- @FunctionTool: isBlocked(applicantName) → BlockedListService
- Returns structured ComplianceResult via responseConformsTo()

### CreditCheckAgent (Agent)
- @FunctionTool: getCreditReport(name) → CreditBureauService
- @FunctionTool: getLoanPolicy() → LoanPolicyService
- Returns structured CreditResult via responseConformsTo()

### LoanApplicationView (View)
- Projects workflow state for list queries + SSE streaming

Step 5

/implement — execute the plan

Walks through tasks in order, generating production-ready code one component at a time. Compiles after each phase, runs tests, stops for approval.

Produces:  working Akka service with tests
Output — CreditCheckAgent.java
@Component(id = "credit-check-agent")
public class CreditCheckAgent extends Agent {

  public Effect<CreditResult> check(CreditCheckRequest request) {
    return effects()
        .systemMessage("""
            You are a credit assessment agent.
            1. Fetch credit report via getCreditReport tool
            2. Fetch loan policy via getLoanPolicy tool
            3. Calculate DTI and LTI ratios
            4. Evaluate against policy thresholds
            """)
        .userMessage("Evaluate: %s, income: %d, loan: %d"
            .formatted(request.applicantName(),
                       request.annualIncome(),
                       request.requestedLoanAmount()))
        .responseConformsTo(CreditResult.class)
        .thenReply();
  }

  @FunctionTool(description = "Fetch credit report")
  public String getCreditReport(String name) {
    var report = creditBureauService.getCreditReport(name);
    return "Score: %d, debt: %d"
        .formatted(report.creditScore(), report.existingDebt());
  }
}
